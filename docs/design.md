# 설계 정본 — food-budget-app

> **이 문서가 소스오브트루스(SSOT).** 모든 설계 결정은 여기에 기록하고, 코드·인프라·문서는 이 문서에서 파생.

## §1. 프로젝트 개요

### 1.1 서비스 한 줄 요약
**월 식비 예산 기반 밀(meal) 플래닝** — 레시피 재료 추출 → 가격 비교 → 예산 계획·추적을 하나의 루프로 연결하는 앱.

### 1.2 핵심 축
**"월 식비 예산 관리"** — 모든 기능은 이 축에 기여하는 방식으로 정렬. 축에서 벗어나는 기능은 P1 이하.

### 1.3 페르소나
20~30대 1인 가구. 예산을 아끼고 싶지만 매번 장보기·요리 계획이 번거로운 사람.

### 1.4 팀
5명 / 8-9주 풀타임.

## §2. 기능 정의 + 우선순위

### P0 (MVP 필수) — "레시피 고르기 → 재료 확인 → 가격 비교 → 예산 기록" 핵심 루프

| # | 기능 | 설명 |
|---|---|---|
| 1 | **레시피 + 재료 추출** | 만개의레시피·냉부에서 사전 크롤링한 레시피 검색, 재료 NER 추출 → 쿠팡 가격 매핑 |
| 2 | **내 냉장고** | 영수증 OCR(종이+이커머스 캡쳐) + 수동입력으로 재고 관리. 유통기한 알림. 보유 재료 기반 레시피 추천 |
| 3 | **장바구니 예산 계산** | 레시피 재료 → 쿠팡 가격 기반 장바구니 생성 → 예산 대비 비용 시각화 |
| 9 | **캘린더 + 엥겔지수** | OCR 자동 연동 + 외식비 수동입력으로 식비 추적. 월별 엥겔지수(식비/소득 비율) 시각화. 장바구니 연동은 확장 |

### P1 (MVP 이후)

| # | 기능 | 설명 |
|---|---|---|
| 0 | **YouTube 레시피북** | 유저가 YouTube URL 제출 → 설명란 우선 + 자막 폴백 → NER → 개인 레시피북(Pinterest 스타일). 레시피북 공유 기능 포함 |
| 6 | **시세 기반 추천** | 쿠팡 가격 이력 기반 "지금 싼 재료" 추천. 최저가 알림 (P0 AI 이상탐지 활용) |
| 7 | **핫딜 + 가격 예측** | 지마켓 타임딜 크롤링 → 관련 레시피 추천. 쿠팡 가격 이력 기반 할인 주기 예측 (AI P1) |
| 5 | **식사 아카이빙** | 식사 사진 + 맛평가 + 가격 기록. 영양소 분석 (DB 룩업) |

### P2 (여유 시 / 데모)

| # | 기능 | 설명 |
|---|---|---|
| 4 | **소셜 피드** | 집밥 피드 공유, 좋아요/댓글 |
| 8 | **블프/추석/명절** | 시즌 테마 변경 → CI/CD 파이프라인 시연용 |

### 향후 확장 (팀 브레인스토밍)
- 핫딜 주기 분석 → 할인 시기 예측 알림
- 레시피별 커스텀 댓글
- 조리이력 + 챗봇 로그 → 개인화 추천·섭취량 평가·비용 경고
- 조리기구별 카테고리 (에어프라이어/전자레인지/오븐)
- 클릭 데이터 기반 실시간 랭킹
- 레시피 업로더 구독·스크랩·좋아요
- 난이도 기반 추천
- 리뷰 감정분석 (AI)
- 남은 재료 나눔 커뮤니티

## §4. AI 모델

> 전부 CPU 학습·서빙 (GTX 1060 3GB → GPU 학습 불가).

### 4.1 AI 라인업

| 모델 | 역할 | 방식 | 우선순위 |
|---|---|---|---|
| **재료 NER** | 레시피 텍스트 → 재료 추출 + 정규화 ("대파 반 단" → 표준 품목코드) | CRF (sklearn-crfsuite) | P0 |
| **최저가 알림** | 쿠팡 가격 이상탐지 (평균 대비 급락 감지) | 통계 (z-score 등) | P0 |
| **신선도 예측** | 냉장고 재고 유통기한 예측 | XGBoost | P1 |
| **레시피 랭킹** | 개인화 레시피 추천 | LightGBM | P1 |
| **할인 주기 예측** | 쿠팡 가격 이력 → "이 상품 다음 할인 예상 시점" | LightGBM 시계열 | P1 |
| **챗봇** | 시세 기반 추천 대화 | 의도분류(FastText) + 템플릿 | P2 |

### 4.2 영양소 분석
AI가 아닌 **DB 룩업**으로 처리. NER 결과(재료명) → 영양성분 테이블 조인 → 7대 영양소 산출.

### 4.3 ChatGPT-moat 서사
"범용 텍스트 파싱은 누구나 할 수 있지만, **한국 식재료 정규화(CRF NER)**는 자체 AI" — 이 정규화된 품목코드가 냉장고 재고 매칭 + 쿠팡 장바구니 가격 비교의 **연결 키**.

### 4.4 YouTube 추출 파이프라인

> 추출 방식 = **Gemini 멀티모달** (유료 API 예외 승인 2026-07-09, `AGENTS.md` 절대제약 3 · 캐싱 §3.4). 모델 조합·미디어 해상도는 **PoC 검증 중** — 파이프라인 정본은 `docs/video-recipe-ai.md`.

```
유저 YouTube URL 제출
  → 사전 필터 (YouTube Data API, 무료: 길이·비요리 카테고리 거절)
  → 교차유저 캐시 조회 (Redis TTL 30일, §3.4) — 히트 시 추출 생략
  → Gemini 멀티모달 추출 → JSON (재료/분량/스텝/타임스탬프)
  → CRF NER 정규화 (재료명 → 표준 품목코드)   ← 자체 AI (moat)
  → ES 유사도 검색 → 쿠팡 상품 매핑
  → 개인 레시피북 저장 (PG)
```

## §5. MSA 서비스 구조

> 상시 서비스 7개 + 배치/파이프라인.

| 서비스 | 역할 |
|---|---|
| **Gateway** | API 라우팅 + JWT 검증 |
| **User** | 인증 (Kakao 소셜 + 자체 이메일/비번) → JWT 발급 |
| **Pantry** | 냉장고 재고 + 유통기한 + OCR 입력 + 안전알림 |
| **Recipe** | 레시피 검색 + 개인 레시피북 + YouTube 추출 + 레시피북 공유 |
| **Price** | 쿠팡 가격 비교 + 가격 이력 + 최저가 알림 + 지마켓 핫딜(P1) |
| **MealPlan** | 예산 밀플래닝 + 장바구니 + 캘린더 + 엥겔지수 |
| **ML Serving** | NER + 이상탐지 + 예측 모델 통합 서빙 |

### 인증 흐름
Kakao 소셜 + 자체 이메일/비밀번호 → 로그인 성공점에서 자체 JWT 통일.

## §6. 기술 스택

### 6.1 확정 스택

| 레이어 | 스택 |
|---|---|
| Frontend | React + Vite + TypeScript, PWA, TanStack Query·Zustand, Tailwind |
| Backend API | **FastAPI** (FastAPI Gateway, PyJWT, SQLAlchemy+Alembic, confluent-kafka, Pydantic) |
| ML 서빙 | **FastAPI** 통합 pod (API와 동일 언어 → 코드 공유) |
| 데이터 파이프라인 | **Python** (confluent-kafka 크롤러/폴러/컨슈머) |
| 저장소 | **PostgreSQL**(OLTP) + **ClickHouse**(쿠팡 가격 시계열) + **Elasticsearch**(레시피+쿠팡 상품 검색, nori) + **Redis** |
| ML | CRF(sklearn-crfsuite)·XGBoost·LightGBM — 전부 CPU |
| Docker | 멀티스테이지, 프론트 nginx:alpine |
| Kubernetes | kubeadm · FastAPI Gateway · HPA+KEDA · Strimzi · ArgoCD |
| AWS | EC2 · Karpenter · ECR · Terraform · GitHub Actions |

### 6.2 재료↔상품 매핑
NER 결과(표준 품목코드) → **Elasticsearch nori 유사도 검색** → 쿠팡 상품 인덱스에서 관련 상품 랭킹. 추후 가중치 튜닝으로 정밀도 향상.

## §7. 데이터 파이프라인

### 7.1 크롤링 파이프라인

```
[스케줄 폴러]
  쿠팡 (일 2회) ──→ Kafka ──→ ClickHouse (가격 이력)
                              ──→ ES (상품 검색 인덱스)

[배치 크롤러]
  만개의레시피 (주 1회) ──→ Kafka ──→ NER 처리 ──→ ES (레시피 인덱스)
  냉부 (주 1회) ──────────→ Kafka ──→ NER 처리 ──→ ES (레시피 인덱스)

[스케줄 폴러 — P1]
  지마켓 타임딜 (일 2회, 11시/20시) ──→ Kafka ──→ PG + Redis (핫딜 알림)
```

### 7.2 온디맨드 파이프라인

```
[유저 요청]
  YouTube URL ──→ 사전필터 + 캐시(Redis) ──→ Gemini 멀티모달 추출 ──→ CRF NER ──→ ES 매핑 ──→ 레시피북 저장 (PG)
  영수증 사진 ──→ OCR (Tesseract) ──→ 파싱 ──→ Pantry DB (PG) + 캘린더 (PG)
```

## §8. 인프라 시나리오

### 8.1 트래픽 스파이크 (멘토 피드백 대응)
- **메인: 일일 피크타임** — 11-12시(점심), 17-18시(저녁) 레시피 검색 + 가격 비교 집중. HPA/KEDA 오토스케일링 시연.
- **보조: 명절 시즌** — 추석/설 전 장보기 시즌 트래픽 집중.

### 8.2 다자간(multi-party) 트래픽 (멘토 피드백 대응)
- **메인: 최저가 알림 fan-out** — 특가 알림 발송 → N명 동시 상품 조회. Redis 캐싱 + HPA 스케일링.
- **보조: 레시피북 공유** — 인기 레시피북에 다수 유저 동시 접근 + 스크랩.

### 8.3 정량적 데이터 (멘토 피드백 대응)
쿠팡 가격 이력, OCR 식비 데이터, 레시피 크롤링 데이터 등 숫자 기반 설계 근거 확보.
