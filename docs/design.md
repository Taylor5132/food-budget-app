# 설계 정본 — food-budget-app

> **이 문서가 소스오브트루스(SSOT).** 모든 설계 결정은 여기에 기록하고, 코드·인프라·문서는 이 문서에서 파생.

## §1. 프로젝트 개요

### 1.1 서비스 한 줄 요약
**월 식비 예산 기반 밀(meal) 플래닝** — 레시피 재료 추출 → 가격 비교 → 예산 계획·추적을 하나의 루프로 연결하는 앱.

### 1.2 핵심 축
**"월 식비 예산 관리"** — 모든 기능은 이 축에 기여하는 방식으로 정렬. 축에서 벗어나는 기능은 P1 이하.

### 1.3 페르소나
20~30대 1인 가구. 예산을 아끼고 싶지만 매번 장보기·요리 계획이 번거로운 사람.

### 1.4 팀
5명 / 8-9주 풀타임.

## §2. 기능 정의 + 우선순위

### P0 (MVP 필수) — "레시피 고르기 → 재료 확인 → 가격 비교 → 예산 기록" 핵심 루프

| # | 기능 | 설명 |
|---|---|---|
| 1 | **레시피 + 재료 추출** | 만개의레시피·냉부에서 사전 크롤링한 레시피 검색, 재료 NER 추출 → 마켓컬리 현재가 매핑 |
| 2 | **내 냉장고** | 영수증 OCR(종이+이커머스 캡쳐) + 수동입력으로 재고 관리. 유통기한 추적·임박 알림(대략 추정). 보유 재료 기반 레시피 추천 |
| 3 | **장바구니 예산 계산** | 레시피 재료 → 마켓컬리 현재가 기반 장바구니 생성(재고 차감·중복 합산) → 예산 대비 비용 시각화 |
| 9 | **캘린더 + 식비 추적** | OCR 자동 연동 + 외식비 수동입력으로 식비 추적. 러닝 트래커(누적/잔여) + 성과지표(안 버린 재료). ~~월별 엥겔지수 → 드롭~~ |
| 10 | **홈 대시보드 + 알림 센터** | (공통·횡단) 예산 잔여·재고·임박·추천·성과 집계 홈 + 유통기한 임박 등 알림 센터 |

### 3.4 크롤링 정책

- 지마켓: 타임딜 식품 카테고리 한정.
- 유저 제출 YouTube 영상 **추출 결과 캐싱**: 재처리 비용 절감 위해 **한시 캐시(Redis TTL 30일, `video_id` 키) 허용**. **영구 공유 저장소 불가**, 유저 레시피북(PG)은 개인용, 출처(원본 영상) 표기. 상세 `docs/video-recipe-ai.md` §6.1.

## §4. AI 모델

> 전부 CPU 학습·서빙 (GTX 1060 3GB → GPU 학습 불가).

### 4.1 AI 라인업

| 모델 | 역할 | 방식 | 우선순위 |
|---|---|---|---|
| **재료 NER** | 레시피 텍스트 → 재료 추출 + 정규화 ("대파 반 단" → 표준 품목코드) | CRF (sklearn-crfsuite) | P0 |
| **최저가 알림** | 마켓컬리 경량 가격 이력 이상탐지 (평균 대비 급락) | 통계 (z-score) | P0 ⚠️ baseline 4주→오탐↑ |
| **신선도 예측** | 냉장고 재고 유통기한 예측 | XGBoost | P1 |
| **레시피 랭킹** | 개인화 레시피 추천 | LightGBM | P1 |
| **챗봇** | 재고·예산 기반 추천 대화 (대화형 어시스턴트는 Gemini 생성 — 외부) | 의도분류(FastText) + 템플릿 | P2 |
| ~~**할인 주기 예측**~~ | ~~가격 이력 → 다음 할인 예상 시점~~ | ~~LightGBM 시계열~~ | **드롭** — 8주로 할인 사이클 부족(예측 불가) |

> **아이디어(미확정, §10 점검):** 유통기한 대략 추정 · 냉동/냉장/실온 자동 분류 · 레시피 자동 태깅 — Gemini 의존/비용 검토 필요.

### 4.2 영양소 분석
AI가 아닌 **DB 룩업**으로 처리. NER 결과(재료명) → 영양성분 테이블 조인 → 7대 영양소 산출.

### 4.3 ChatGPT-moat 서사
"범용 텍스트 파싱은 누구나 할 수 있지만, **한국 식재료 정규화(CRF NER)**는 자체 AI" — 이 정규화된 품목코드가 냉장고 재고 매칭 + 마켓컬리 장바구니 가격 비교의 **연결 키** (정본 = §6.3 표준 품목 마스터).

### 4.4 YouTube 추출 파이프라인

> 추출 방식 = **Gemini 멀티모달** (유료 API 예외 승인 2026-07-09, `AGENTS.md` 절대제약 3 · 캐싱 §3.4). 모델 조합·미디어 해상도는 **PoC 검증 중** — 파이프라인 정본은 `docs/video-recipe-ai.md`.

```
유저 YouTube URL 제출
  → 사전 필터 (YouTube Data API, 무료: 길이·비요리 카테고리 거절)
  → 교차유저 캐시 조회 (Redis TTL 30일, §3.4) — 히트 시 추출 생략
  → Gemini 멀티모달 추출 → JSON (재료/분량/스텝/타임스탬프)
  → CRF NER 정규화 (재료명 → 표준 품목코드)   ← 자체 AI (moat)
  → ES 유사도 검색 → 마켓컬리 상품 매핑
  → 개인 레시피북 저장 (PG)
```

### 6.2 재료↔상품 매핑
NER 결과(표준 품목코드) → **Elasticsearch nori 유사도 검색** → 마켓컬리 상품 인덱스에서 관련 상품 랭킹. 추후 가중치 튜닝으로 정밀도 향상. (품목코드 정본 = §6.3)

### 6.3 표준 품목 마스터 (연동 정본) — 체인의 조인 허브

> data-validation §3.1 최우선 블로커 해소. CRF·nori·영양·재고·가격이 매달릴 canonical 코드공간을 **선행 산출물**로 정의. **오프라인 1회 구축·유지.**

- **앵커**: 식약처 식품코드(국가표준식품성분표, data.go.kr 15100064/65)로 앵커링한 **자체 canonical 품목 사전** → 영양 7대 조인 공짜.
- **스코프**: 실사용 재료 **~300~500 품목** (식약처 원물 ~4,140 필터 → 팀 큐레이션). = 마켓컬리 핵심 SKU 폴링 집합.
- **마스터 스키마(1행)**: `item_code`(자체 PK) · 대표명 · 식약처_식품코드 · 분류(식약처 계층) · 동의어[] · 약어[] · 기본단위·용량규칙 · 대표_컬리_상품no.
- **명칭공간별 정규화기 → item_code**:
  - 레시피 재료명 → CRF NER + 동의어 사전/nori
  - 마켓컬리 상품명 → nori + `volume` 구조화(§8.2 실측) → item_code + 100g단가
  - 영수증 약어 → 파서 + **HITL 확인 UI** (자동 수렴 불가, data-validation §6 전제)
  - 식약처 식품명 → 구축 시 이름조인(= 앵커)
- **사전**: 동의어=식약처 이명/대표식품명 + 수동 시드 · 약어=유저 영수증 쌓이며 HITL 점진 부트스트랩(콜드스타트) · 용량=기본단위 환산 규칙.
- **유지**: 마스터 오너 1인 지정(§10 역할분담 연동) + 신규 재료 온디맨드 추가 + 식약처 원물 분기 재동기화.
- 🧪 **잔여 미검증**: 식약처 대표명 ↔ 구어 재료명 라이브 매칭률(대파·무 등) — PoC 전 확정 불가 (data-validation §7.1).

## §8. 인프라 시나리오

### 8.1 트래픽 스파이크 (멘토 피드백 대응)
- **메인: 일일 피크타임** — 11-12시(점심), 17-18시(저녁) 레시피 검색 + 가격 조회 집중. HPA/KEDA 오토스케일링 시연. *(Kafka/KEDA 서사 = 대량 스트리밍이 아닌 이벤트-드리븐 디커플링 + 예측가능 버스트)*
- **보조: 명절 시즌** — 추석/설 전 장보기 시즌 트래픽 집중.

### 8.2 다자간(multi-party) 트래픽 (멘토 피드백 대응)
- **메인: 최저가 알림 fan-out** — 마켓컬리 이상탐지 급락 감지 → N명 동시 상품 조회. Redis 캐싱 + KEDA(알림 큐 버스트)/HPA. *(통제 소스=마켓컬리 폴링)*
- **보조: 레시피북 공유 / 지마켓 핫딜** — 인기 레시피북 다수 동접 + 스크랩, 타임딜 알림.

### 8.3 정량적 데이터 (멘토 피드백 대응)
DAU 500(가정) 종이 추정: PG 저장 **수십만 행·수백 MB**(가격 이력 ~56K + OCR 식비 ~56K + 재고·레시피), 레시피 검색·가격 조회 **~2,500 req/일**(11-12·17-18 피크 ~30% 집중), 최저가 알림 fan-out 버스트. → **단일 PG 충분**(ClickHouse 불필요 재확인). *(쿠팡 가격 이력 축 드롭 → 마켓컬리 경량 이력으로 대체)*
